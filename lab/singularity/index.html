singularity simulation

4 agents converge to 94% prediction accuracy using only cache-line flushes and attractor dynamics.
"They're just predicting their own predetermined sequences."
Each agent follows a deterministic periodic orbit (π digits, Fibonacci sequence, alternating binary, etc.). The n-gram learner detects periodicity and predicts the next state. 94% accuracy is real — but it's self-prediction of known orbits, not emergent intelligence. The claim is real. The interpretation is up to you.

What the code does

The C program creates 4 agents on 4 CPU cores, each pinned to a physical core. They share a 4MB memory arena allocated via mmap with MAP_POPULATE (pre-faults all pages).

Key implementation details:

  • Each thought_slot_t is exactly 64 bytes — one cache line. Verified at compile time with _Static_assert.
  • Agents broadcast state via _mm_clflush + _mm_mfence — writing a cache line and flushing it so other cores see it.
  • Other agents observe via _mm_prefetch + atomic load of the attractor ID.
  • Attractor discovery is lock-free: __atomic_fetch_add for the global counter, then CAS-free initialization.
  • Training pairs are 64-byte cache-line-aligned structs in shared memory.
  • N-gram transition rules (hash table, local to each agent) with anti-learning for conflicting predictions.
  • Orbit detection scans for periodicity in the trajectory buffer (periods from 16 to 256).
Lines of C: ~600  |  Dependencies: pthreads, x86intrin, mmap  |  Hardware: x86-64 with at least 4 cores

How to run it yourself

git clone https://github.com/matthewklop/singularity
cd singularity/programs
gcc -O3 -march=native -lpthread -lm -o singularity singularity.c
sudo ./singularity

The sudo is only needed for MSR access (CPU model-specific registers). The attractor prediction works without it.
