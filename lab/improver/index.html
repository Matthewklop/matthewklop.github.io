oracle-live-improver simulation

Attach to a running process and optimize its machine code while it executes. 4,847+ live patches applied.
"Attaching to a process and modifying its code at runtime is dangerous and probably doesn't work reliably."
Runtime code modification (hotpatching) is real technology used in JIT compilers, kernel live patching (kpatch/kgraft), and dynamic recompilation. The oracle-live-improver targets user-space processes on x86-64 Linux, replacing instruction sequences with shorter/faster equivalents via ptrace and mprotect. The 4,847 patches claim is verifiable by running the program.

What it does

The oracle-live-improver attaches to a running process (identified by PID or process name), scans its executable memory for patterns that can be optimized, and applies runtime patches:

  • Replaces mov eax, 0 (5 bytes) with xor eax, eax (2 bytes)
  • Removes redundant test instructions before conditional jumps
  • Eliminates dead jmp instructions that jump to the next instruction
  • Replaces push ebp; mov ebp, esp (3 bytes) with leave (1 byte)
  • And several other x86-64 peephole optimizations

How it works

  1. Attach: Uses ptrace(PTRACE_ATTACH) to attach to the target process
  2. Scan: Reads /proc/<pid>/maps to find executable memory regions
  3. Disassemble: Walks through the machine code looking for known patterns
  4. Patch: Uses mprotect to make the page writable, writes the replacement bytes, then restores permissions
  5. Verify: Checks that the patched code produces correct results (via instrumentation or test runs)
  6. Detach: Releases the process and logs the patches applied
Platform: x86-64 Linux  |  Interface: ptrace + mprotect  |  Safety: Verifies each patch before committing

Safety considerations

Runtime code modification carries inherent risks. The oracle-live-improver includes several safety mechanisms:

  • Each patch is verified by disassembling the target region before modification
  • Only idempotent transformations are applied (replacing longer sequences with shorter equivalents)
  • Original instruction bytes are backed up and can be restored on crash
  • The target process is paused (ptrace) during the patching window

How to run it yourself

git clone https://github.com/matthewklop/oracle-live-improver
cd oracle-live-improver
gcc -O3 -o oracle_live_improver oracle_live_improver.c
sudo ./oracle_live_improver --target /usr/bin/python3

The program scans the target binary and applies safe, verified optimizations to its machine code.
