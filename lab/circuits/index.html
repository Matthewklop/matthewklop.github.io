oracle-circuits simulation

Describe a circuit in English. Get transistor count, delay, and truth table at 14nm.
"It's a lookup table with hardcoded values, not real silicon design."
That's fair. The program matches descriptions against known circuit patterns and returns precomputed specs. It doesn't design novel circuits from scratch. But the specs (transistor counts, delays at 14nm) are accurate for the circuits listed. It's a reference tool, not a synthesis engine.

What it does

Given an English description like "half adder" or "D flip-flop", the program returns:

  • Gate count and type breakdown
  • Total transistor count
  • Propagation delay at 14nm process node
  • ASCII schematic showing the circuit structure
  • Truth table simulation for all input combinations
  • Power and energy estimates

How it works

The C code uses a simple pattern-matching approach:

  • Predefined CircuitPattern structs with name, description, gate count, transistor count, delay, and schematic lines
  • User input is lowercased and matched against circuit names via strstr
  • Gate-level delay values are looked up from a gate info table (NOT=1.59ps, NAND=3.18ps, XOR=9.53ps at 14nm)
  • Truth tables are generated by simple logic simulation (e.g., sum = a ^ b ^ cin for full adder)
  • Power estimates are calculated from gate-level power data
Circuits available: 7  |  Lines of C: ~300  |  Dependencies: libm only

Limitations

The program does not:

  • Design novel circuits it hasn't seen before
  • Perform SPICE-level transistor simulation
  • Generate layouts or netlists
  • Optimize for power, area, or timing

It is essentially an interactive reference for common digital circuit parameters. Useful for education and quick estimates, not for tapeout.

How to run it yourself

git clone https://github.com/matthewklop/oracle-circuits
cd oracle-circuits
gcc -O3 -o oracle_circuit oracle_circuit.c -lm
./oracle_circuit "half adder"
./oracle_circuit list
