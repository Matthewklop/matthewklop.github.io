A programming language where every token fits in 16 bytes. Compiles to C.
"It's just a toy language that generates C code. Nothing special."
The Stone grammar is designed so the cascade LLM can tokenize it at 30M tokens/sec — no AST construction, no symbol table, no parsing step. The token stream is the grammar. The compiler works. Write Stone, get C.

What is Stone?

="font-size: 0.85rem; color: var(--dim); margin-bottom: 0.5rem"> Stone is a minimal stack-based language designed for maximum token throughput in LLM-based compilation pipelines.

  • Every token is ≤16 characters — fits in a
    • single GPT-4o BPE token
  • Stack-based evaluation — no parentheses, commas, or semicolons
  • Functions are 64-byte aligned for cache-line-friendly dispatch
  • No AST
    • — the token stream is the parse tree
  • No symbol table — values are addressed by stack position
  • No parser — the linear tokenizer is the frontend

This means the cascade can compile Stone to C in a single forward pass with zero backpressure. Every 16-byte token maps directly to a C construct.

Multi-Target Transpiler

The Stone compiler targets multiple backends from the same IR-less frontend:

C
Direct struct-to-struct translation. GCC/Clang compatible. No runtime.
Python
Generates flat Python with native stack emulation. No imports.
JavaScript
ES module output. Everything is let. Runs in any browser.
Lua
LuaJIT-friendly output. Tables as stacks. No metamagic.

How to Run

Clone the repo and use the Makefile:

git clone https://github.com/matthewklop/stone.git cd stone && make ./bin/stone examples/fib.st > out.c gcc -O2 out.c -o fib && ./fib

Requirements: GCC or Clang, Make, a POSIX shell.
